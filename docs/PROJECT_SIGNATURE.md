## Propósito

Disponibilizar um aplicativo Flutter para apoiar operações de armazém, com integração à API Laravel do WMS e suporte progressivo a operação offline/local storage.

## Stack

- Flutter
- Dart
- Android
- API Laravel
- SQLite local
- Dio e HTTP client legado em transição
- SharedPreferences
- Flutter Secure Storage
- Connectivity Plus

## Padrão de Engenharia

O projeto deve seguir desenvolvimento orientado por análise prévia de regra de negócio, arquitetura, dados, segurança e validação. Mudanças em contratos de API, estrutura de pastas, banco local ou suporte offline devem ser justificadas e documentadas.
